11. 2026-04-13｜ServiceNow 如何讓客戶盡情享受 AI 低谷

全文翻譯（繁體中文・香港）

【翻譯失敗，保留原文】ServiceNow's latest product announcements show how hardcore the company has become about embedding AI across its go-to-market strategy.

ServiceNow 高級副總裁 John Aisien 負責監督前沿部署工程以及與 Anthropic 和 Microsoft 等公司的策略合作夥伴關係，他在接受 The Register 採訪時概述了這些主要舉措。

「人工智慧現在已融入我們向目標市場提供的每一個產品中，」他說。

首先，ServiceNow 圍繞著三個層級的人工智慧功能重新調整了定價。客戶現在可以選擇與其人工智慧成熟度相對應的定價水平。

輔助人工智慧總結數據或生成內容。任務自動化從頭到尾處理離散作業。透過完全角色自動化，人工智慧工作流程可以在最少的人工監督下自主運作。

Aisien 表示：“無論組織是否準備好使用輔助人工智慧、人工智慧自動化，還是使用自主人工智慧的全面角色自動化，我們都引入了一種新的打包模型，使他們能夠訪問分為這三個類別的整個 ServiceNow 價格表。”

Aisien 將第二個公告 Build Agent SDK 描述為「真正的遊戲規則改變者」。該工具旨在讓任何開發人員可以從他們已經使用的任何編碼環境創建或修改 ServiceNow 應用程序，無論是 GitHub Copilot、Cursor、Codex 還是各種自然語言「vibe 編碼」工具。

「我們希望世界上有盡可能多的開發人員——無論開發形式因素和他們獲得的開發經驗如何——我們希望這些開發人員能夠超快速地建立 ServiceNow 工作負載並運行這些工作負載，」他說。 “構建代理 SDK 是實現這一承諾的技術體現。”

Aisien 描述了賓州的一家生命科學公司，該公司擁有約 8,000 名開發人員，其中 650 名開發人員使用 ServiceNow Studio 進行開發。這使得超過 7,000 人不這樣做——不是因為他們缺乏興趣，而是因為開發人員往往忠於他們喜歡的工具。 Build Agent SDK 旨在滿足所有已經工作的開發人員的需求。

ServiceNow 的 Context Engine 是另一項新產品，它是一種企業旋轉，類似於用戶在 Netflix 和 Amazon 等消費應用程式中看到的個人化——系統可以從每次點擊、懸停和暫停中學習，以提供客製化體驗。 Aisien 表示，企業軟體在採用該技術方面已經落後了。

「歷史背景和持續學習，這已經成為消費者應用程式的常態，」他告訴我們。 「企業軟體確實還沒有學會這種陳舊的模式。人工智慧現在正在迫使這種結構成為現實。為什麼？因為人工智慧是軟體，最終需要數據來做出自主決策。”

他將情境引擎描述為使自主人工智慧在企業規模上發揮作用的重要基礎設施。他稱之為「企業人工智慧實現現實所需的最後一哩技術之一」。

「後端的上下文引擎會說『我將根據他們提出的問題、互動範式，然後根據他們在後端的身份和權限來確定這個人想要對數位資產做什麼的意圖，』」Aisien 說。

ServiceNow 還推出了企業服務管理套件，這是一個針對擁有約 1,000 至 5,000 名員工的公司的新捆綁包。它將 IT、人力資源、供應鏈和財務服務工作流程整合到一個產品中，並將其與人工智慧驅動的實施代理配對，從而大大簡化了部署。

Aisien 表示，傳統的 ServiceNow 實施可能需要六個月或更長時間，而新方法可以將其壓縮到大約 30 天。他說，大部分時間都來自組織後勤，而不是配置。

「使用者介面的實際時間可能是一兩天，」他說。
