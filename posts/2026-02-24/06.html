⌨️ Claude Code 新功能好用定係噱頭？資深開發者親身體驗分享

連登 Tech 版熱議爆發

自 Anthropic 宣佈 Claude Code 多代理並行協作功能升級後，連登 Tech 版隨即出現多條相關討論帖，其中一篇題為「Claude Code 新功能係真係好用定係噱頭？有冇人試過？」的帖文在數小時內累積超過 800 個回覆，成為近期連登科技版最熱門的討論之一。帖文原著自稱是一名在港資科技公司工作超過十年的資深後端工程師，花費兩天時間對新功能進行了全面測試，並以詳盡的篇幅分享了自己的親身體驗和深入見解，引發了大量同行的共鳴和討論。

正面體驗：大型代碼庫重構的驚喜

帖文原著首先分享了令他印象最深刻的正面體驗：他將公司一個擁有超過 80 個微服務的 Java 代碼庫交給 Claude Code 多代理模式進行全面的依賴版本升級和安全漏洞修補任務。系統自動將任務分配給六個並行 Agent，每個 Agent 負責約 10-15 個相關聯的微服務，整個升級過程在不到三小時內完成，相比以往人工逐一處理需要超過兩週的工作量，效率提升令他「嚇一跳」。他特別讚揚了系統在自動生成升級說明文檔和測試覆蓋率報告方面的表現，稱這些輔助功能比主要功能更令他感到驚喜。

批評觀點：複雜業務邏輯的明顯不足

然而，帖文原著亦坦誠指出了他在測試中發現的明顯局限。當他嘗試讓多代理系統處理涉及複雜跨服務業務邏輯的功能開發任務時，系統明顯力不從心——各 Agent 對同一業務概念的理解存在分歧，導致生成的代碼在整合後出現大量邏輯矛盾，最終需要資深工程師花費大量時間進行人工修正。他總結道：「對於有明確規則和模式的機械性工作，Claude Code 升級版確實好用；但涉及需要深入理解業務背景的複雜邏輯，佢仲係好傻，唔好太依賴佢。」

其他網民的補充與辯論

帖文引發了大量其他開發者的回覆，形成了豐富的討論生態。一位自稱在銀行業從事核心系統開發的網民表示，他的測試結論更為悲觀，認為新功能對金融系統的嚴格合規要求完全不適用；另一位初創公司的前端工程師則分享了截然不同的正面體驗，指新功能在 React 組件重構和 TypeScript 類型系統整理方面「神乎其技」，大幅提升了他的個人開發效率。討論中亦有人提出，AI 工具的效用很大程度上取決於工程師提問和引導的技巧，缺乏這種「AI 協作素養」的用戶往往難以發揮新功能的真正潛力。

業界觀察與結語

這場連登討論折射出香港本地開發者群體對 AI 輔助編程工具日趨成熟但仍持批判性的態度。與早年對 AI 工具的盲目追捧不同，現時的本港 IT 從業員普遍能夠更客觀地評估 AI 工具的實際應用價值，清楚區分其適用場景和局限所在。帖文原著最後補充道，儘管對新功能有保留，他仍認為多代理協作是 AI 輔助開發的正確演進方向，期待 Anthropic 在未來版本中能夠顯著提升系統對業務邏輯的理解深度，令這一功能從「好噱頭的高效工具」真正蛻變為「可信賴的開發夥伴」。
