01. 2026-04-29｜Hashicorp 聯合創始人 Mitchell Hashimoto 表示 GitHub“不再是認真工作的地方”

全文翻譯（繁體中文・香港）

Hashicorp 聯合創始人 Mitchell Hashimoto 認為 GitHub 非常不穩定，“不再是認真工作的地方”，因此將把他目前的專案轉移到其他地方。

Hashimoto 目前最喜歡的作品是 Ghostty，這是一款終端模擬器，The Register 稱讚其速度快，並為非常成熟的軟體類別添加了「一些有趣的新功能」。

我想要發佈軟體，但它不希望我發佈軟體

與許多開發人員一樣，橋本使用 GitHub 來開發該項目，並在周二的帖子中宣稱自己是該項目的粉絲。

「我是 GitHub 用戶 1299，於 2008 年 2 月加入，」他寫道，從那時起幾乎每天都使用它。

「GitHub 是讓我最快樂的地方，」他透露，並且「總是為它騰出時間」——即使是在他的蜜月期間，因為這項服務是「我歷來最快樂、最想去的地方」。

「有些人注定會滾動社交媒體。自從這個詞出現之前，我就一直在滾動 GitHub 問題，」他承認。 「假期時，我會在 GitHub 上收藏我想研究的不同項目的書籤。不僅僅是源代碼，還有 OSS 流程、其他維護人員如何應對困難情況。等等。不管你信不信，我喜歡這個。”

這些天他越來越不喜歡 GitHub 了。

「我對此感到憤怒。我傷害了人們的感情。我一直在猛烈抨擊。因為 GitHub 每天都在讓我失望，而且這是針對個人的。這是非理性的個人行為，」他寫道。

他憤怒的原因是服務變得不可靠。

「在過去的一個月裡，我一直在寫日記，在 GitHub 中斷對我的工作能力產生負面影響的每個日期旁邊都畫上‘X’，」他寫道。 “幾乎每天都有一個‘X’。在我寫這篇文章的那天，我已經有約 2 個小時無法進行任何 PR 審查，因為 GitHub Actions 出現了中斷。”

Hashimoto 是在 4 月 28 日事件發生前幾天寫下這篇文章的，該事件導致拉取請求由於 Elasticsearch 的問題而無法完成。

類似的事件意味著橋本認為 GitHub “如果它每天讓你花上幾個小時，就不再是一個可以認真工作的地方了。”

「這對我來說不再是一個有趣的地方了，」他感嘆道。 「我想在那裡，但它不希望我在那裡。我想完成工作，但它不希望我完成工作。我想發佈軟體，但它不希望我發佈軟體。”

開發人員表示，他希望 GitHub 得到改進，但“我也想編碼。但我不能再用 GitHub 編碼了。很抱歉。18 年過去了，我必須離開了。”

如果 GitHub 能夠提供“真正的結果和改進，而不是言語和承諾”，他願意回歸。
