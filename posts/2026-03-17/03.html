03. 2026-03-17｜AWS S3 迎來 20 歲生日並達到“數百艾字節”

全文翻譯（繁體中文・香港）

週六，亞馬遜網路服務慶祝了其簡單儲存服務 (S3) 的 20 歲生日，並透露了有關該服務的一些小秘密。

S3 於 2006 年 3 月 14 日推出，根據主要開發倡導者 Sébastien Stormacq 在亞馬遜的生日派對上發布的帖子，最初提供「跨越三個資料中心的 15 個機架中約 400 個儲存節點的約 1 PB 總儲存容量，總頻寬為 15 Gbps」。

今天，該帖子指出，S3“在全球 39 個 AWS 區域的 123 個可用區中存儲了超過 500 萬億個對象，每秒處理超過 2 億個請求，涵蓋數百艾字節的數據。”

AWS 使用以下奇怪的指標來描述 S3 的規模：“如果將數千萬個 S3 硬碟全部堆疊在一起，它們將到達國際太空站並幾乎返回。”

大多數 3.5 吋硬碟高度為 26 毫米，國際太空站的軌道距離地球約 400 公里。我們知道 AWS 購買客製化硬件，但假設該公司堅持使用標準外形規格，我們認為 S3 使用約 2.76 億個硬碟。 *

在這篇文章中，Stormacq 將 S3 的一致性列為該服務最引人注目的成就。

「你在 2006 年為 S3 編寫的程式碼至今仍然有效，沒有任何變化，」他寫道。 「您的資料經歷了 20 年的創新和技術進步。我們透過多代磁碟和儲存系統遷移了基礎設施。處理請求的所有程式碼都已重寫。但您 20 年前儲存的資料今天仍然可用，並且我們保持了完整的 API 向後相容性。”

該帖子還指出，S3 API“已被整個存儲行業採用並用作參考點”，並且“多家供應商現在提供 S3 兼容的存儲工具和系統，實現相同的 API 模式和約定。”

下載量超過 130TB，因此 Backblaze 將其分解為 200GB 的物件。

這也是一個非常重大的成就。在 AWS 推出 S3 時，本刊記者編輯了一份以存儲為中心的澳大利亞出版物，幾個月內，初創備份供應商開始使用該服務作為新的存儲層，使他們能夠大規模處理數據，而使用當時的本地技術來存儲或保護這些數據的成本高昂。因此，S3 為資料保護創造了新的可能性。

雲端儲存的可用性也具有文化足跡：Netflix 和 Spotify 是眾所周知的 S3 用戶，並使用該服務來快速擴展。這兩家串流媒體巨頭都樹立了影片和音樂產業其他公司效仿的榜樣。

S3 也引起了許多問題，尤其是最初將服務上的所有資源開放給公眾存取的特殊決定，除非使用者限制存取。一些 S3 用戶認為默默無聞會帶來安全性，但一旦犯罪分子尋找開放的 S3 儲存桶，他們就會發現數千個不安全的雲端儲存設定。

S3 也曾遭遇過宕機，最臭名昭著的是 2017 年，當時亞馬遜有問題的 US-EAST-1 地區出現問題，導致一些主要網站癱瘓數小時。

亞馬遜的生日貼文強調了積極的一面，特別是 S3 的 11 個 9 (99.999999999%) 耐用性和無損操作。

這封信透露了一些有關 AWS 如何提供這種可靠性的資訊。

Stormacq 寫道：“S3 持久性的核心是一個微服務系統，它會持續檢查整個隊列中的每個位元組。” “這些審計服務會檢查數據，並在檢測到退化跡象時自動觸發修復系統。”
