16. 2026-03-18｜微軟承諾在 Fabric 上推出一體化資料庫整理中心

全文翻譯（繁體中文・香港）

微軟推出了一款資料庫管理工具，承諾將協助使用者管理共享單一 SQL 引擎的多個資料庫。

借助供應商的 Fabric 資料平台，Database Hub 承諾為工程師提供一個單一位置來管理一系列常見資料庫服務，包括 Azure SQL Server、多模型系統 Azure Cosmos DB、Azure Database for PostgreSQL、Azure Arc 支援的 SQL Server、Azure Database for MySQL 以及其他 Fabric 服務。

微軟表示，「統一資料庫管理體驗」現已開放，將有助於管理本地、PaaS 和 SaaS 上的系統。微軟公司資料庫副總裁 Shireesh Thota 表示：“透過將資料庫原生內建於 Microsoft Fabric，我們正在幫助客戶實現 SQL 現代化，統一他們的資料資產，並更快、更有信心地建立人工智慧原生應用程式。”

他表示，Database Hub 將使那些「組織通常透過分散的工具、入口網站和管理體驗來跨邊緣、PaaS 和 SaaS 環境管理關係型資料庫和 NoSQL 資料庫的使用者」受益。

將如此多的資料庫控制集中在一個地方後，微軟計劃在上面加入一些人工智慧，聲稱這將節省時間並提高效能。

「資料庫中心引入了一種代理輔助、人機互動的資料庫管理方法。智慧代理人不斷對整個範圍內的訊號進行推理，以揭示發生了什麼變化，解釋其重要性，並指導團隊下一步該做什麼，」Thota 說。

技術團隊是否會相信人工智慧代理的「推理」是另一回事。

Thota 表示，微軟的 LLM 工具 Copilot 也將提供見解，幫助團隊快速了解其資料庫資產中發生的情況及其原因。

“整體健康狀況視圖、常見績效類別和趨勢分析提供了跨服務的一致信號，使營運和開發團隊能夠更有信心地從洞察轉向行動。”

The Register 詢問微軟其 Hub 是否有助於調整資料庫，這要求系統建構者權衡大量選擇以優化效能。它們可能包括系統旋鈕（例如運行時參數和記憶體快取策略）、實體設計（例如資料結構或索引類型）、控制資料庫如何執行查詢的查詢調優選項，最後是涉及何時升級軟體或硬體的長期決策的生命週期管理。

去年，卡內基美隆大學資料庫小組發表了一篇論文，顯示向量嵌入演算法可以將常見 PostgreSQL 資料庫服務的預設值效能提高二到十倍。使用單獨的“LLM 助推器”，運行該協議的時間可以從大約 12 小時縮短到大約 50 分鐘。

在過去的幾年裡，微軟展現了推廣一系列資料庫系統以及自己的 SQL Server 的興趣。

去年，微軟推出了一個基於關係型 PostgreSQL 後端構建的文檔資料庫平台，並在此過程中開發了 RDBMS 的兩個開源擴展，這標誌著它已經取得了多大的進步。

去年 11 月，微軟宣布推出分散式 PostgreSQL 資料庫服務，旨在與其他超大規模系統和第三方 RDBMS（例如 CockroachDB 和 YugabyteDB）競爭。

微軟尚未透露這兩個系統是否會納入資料庫中心管理。

其他資料分析和機器學習供應商，例如 Databricks 和 Snowflake，已在其平台內發布了事務資料庫服務。去年，Snowflake 在其 AI 資料環境中推出了 PostgreSQL 資料庫即服務，將事務工作負載與分析和 AI 一起置於一組治理規則下。在收購提供無伺服器 PostgreSQL 架構的 Neon 後，Databricks 宣布推出其服務 Lakebase。 ®
