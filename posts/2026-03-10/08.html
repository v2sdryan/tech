08. 2026-03-09｜Anthropic 推出自動化 Code Review 工具

全文翻譯（繁體中文・香港）

Anthropic 引入了一種更廣泛且昂貴的方式來審查託管儲存庫中的原始程式碼，其中許多儲存庫已經包含大量人工智慧生成的程式碼。

程式碼審查是一項面向團隊和企業客戶的新服務，它促使多個代理共同搜尋程式碼儲存庫，以捕獲未識別的錯誤。

該公司的 Claude 模型已經能夠根據需要進行程式碼審查——您可以透過讓 Claude 審查自己的工作來了解很多有關 AI 生成程式碼的品質的資訊。 AI 業務還提供了 Claude Code GitHub Action，可以作為 CI/CD 管道的一部分自動啟動程式碼審查。

程式碼審查會做更多的事情，但代價也更大。

該公司在其文件中解釋道：“Code Review 分析您的 GitHub 拉取請求，並將結果作為內聯註釋發佈在發現問題的代碼行上。” “一組專門的代理在完整代碼庫的上下文中檢查代碼更改，查找邏輯錯誤、安全漏洞、損壞的邊緣情況和微妙的回歸。”

你說的是一支專門特工艦隊？聽起來它可能會在推理過程中燃燒大量代幣。事實確實如此。正如 Anthropic 所觀察到的，程式碼審查比現有方法更注重深度。

該公司表示：“評論按代幣使用情況計費，平均價格為 15-25 美元，並根據 PR [拉取請求] 的規模和複雜性進行調整。”

這是每個拉取請求。作為比較，Code Rabbit 提供基於人工智慧的程式碼審查，每月收費 24 美元。

程式碼審查也不是很快。據 Anthropic 稱，雖然所需的時間因拉取請求的大小而異，但審核平均需要大約 20 分鐘才能完成。

考慮到所需的時間和計費率，問題變成每小時支付 60 美元進行程式碼審查是否會產生類似或更好的結果。

儘管如此，人工智慧產業堅稱其工程師使用程式碼審查已經看到了積極的結果，這項發現得到了一些研究的支持，但並非在所有情況下都是如此。

Anthropic 報告稱，它已在內部使用 Code Review 幾個月並取得了相當大的成功。該公司聲稱，對於包含 1,000 多個更改行的大型拉取請求，84% 的自動審查會發現一些值得注意的內容，平均發現 7.5 個問題。對於少於 50 行的小型 Pull 請求，31% 的人會收到評論，平均有 0.5 個問題。

人類開發人員拒絕克勞德發現的問題的比例不到百分之一。

測試 Code Review 的客戶已經看到了一些好處。當 TrueNAS 開始對其開源中間件進行 ZFS 加密重構時，AI 審查服務發現了相鄰程式碼中的一個錯誤，該錯誤存在類型不匹配會在同步操作期間擦除加密金鑰快取的風險。

Anthropic 聲稱，在一個涉及內部程式碼的實例中，Code Review 發現了對生產服務的看似無害的單行更改，該更改可能會破壞服務的身份驗證機制。

「這個問題在合併之前就被修復了，工程師後來表示，他們自己不會發現這個問題，」人工智慧公司表示。

在足夠大、能夠負擔得起人工智慧工具的組織中，軟體開發人員是否會再次單獨工作值得懷疑。 ®
