19. 2026-03-09｜Chrome 擴充功能轉手後變惡意程式，涉注入代碼與竊資

全文翻譯（繁體中文・香港）

兩個 Google Chrome 擴充功能在看似所有權轉讓的情況下變成了惡意軟體，為攻擊者提供了一種向下游客戶推送惡意軟體、注入任意程式碼並獲取敏感資料的方法。

下面列出了有問題的擴充程序，它們最初都與名為“akshayanuonline@gmail.com”（BuildMelon）的開發人員相關 -

雖然 QuickLens 不再可以從 Chrome Web Store 下載，但截至撰寫本文時，ShotBird 仍然可以存取。 ShotBird 最初於 2024 年 11 月推出，其開發者 Akshay Anu S (@AkshayAnuOnline) 在 X 上聲稱該擴展適合“創建專業的、類似工作室的視覺效果”，並且所有處理都在本地進行。

根據 monxresearch-sec 發布的研究，該瀏覽器外掛程式在 2025 年 1 月收到了「精選」標誌，然後在上個月的某個時候被傳遞給其他開發人員（「loraprice198865@gmail.com」）。

類似地，Annex Security 的 John Tuckner 表示，QuickLens 於 2025 年 10 月 11 日在 ExtensionHub 上上市出售，發布僅兩天后，「akshayanuonline@gmail.com」就發布了。 2026 年 2 月 1 日，該擴充功能的擁有者在 Chrome 應用程式商店清單頁面上變更為「support@doodlebuggle.top」。

QuickLens 於 2026 年 2 月 17 日引入的惡意更新保留了原始功能，但引入了從每個 HTTP 回應中剝離安全標頭（例如 X-Frame-Options）的功能，允許注入網頁的惡意腳本向其他網域發出任意請求，繞過內容安全策略 (CSP) 保護。

此外，該擴充功能還包含用於識別使用者所在國家/地區、偵測瀏覽器和作業系統以及每五分鐘輪詢外部伺服器以接收 JavaScript 的程式碼，該 JavaScript 儲存在瀏覽器的本機儲存中，並透過新增隱藏的 1×1 GIF 元素並將 JavaScript 字串設定為其「onload」屬性，在每次頁面載入時執行。這反過來又會導致惡意程式碼在圖像載入後被執行。

「實際的惡意程式碼永遠不會出現在擴展的源文件中，」塔克納解釋道。 “靜態分析顯示了一個創建圖像元素的函數。就是這樣。有效負載從 C2 傳遞並存儲在本地存儲中 - 它們僅在運行時存在。”

monxresearch-sec 對 ShotBird 擴充功能進行的類似分析發現，使用直接回呼來傳遞 JavaScript 程式碼，而不是建立 1x1 像素映像來觸發執行。 JavaScript 被設計為顯示虛假的 Google Chrome 瀏覽器更新提示，點擊為使用者提供 ClickFix 樣式頁面以開啟 Windows 執行對話方塊、啟動「cmd.exe」並貼上 PowerShell 命令，從而在 Windows 主機上下載名為「googleupdate.exe」的可執行檔。

然後，惡意軟體會繼續掛鉤輸入、文字區域、選擇 HTML 元素，並捕獲受害者輸入的任何資料。這可能包括憑證、PIN、卡片詳細資料、令牌和政府識別碼。它還可以竊取 Chrome 網路瀏覽器中儲存的數據，例如密碼、瀏覽歷史記錄和擴充功能相關資訊。

研究人員表示：「這是一個兩階段的濫用鏈：擴展端遠端瀏覽器控制加上透過虛假更新進行的主機級執行樞紐。」「結果是瀏覽器中的高風險資料暴露，並在至少一個受影響的系統上確認主機端腳本執行。實際上，這將僅瀏覽器濫用的影響提升到可能的憑證盜竊和更廣泛的端點危害。」

根據評估，考慮到使用相同的命令和控制 (C2) 架構模式、注入瀏覽上下文的 ClickFix 誘餌以及將所有權轉移作為感染媒介，這兩個擴展的攻擊背後是同一威脅參與者，並且正在並行操作它們。

有趣的是，最初的擴充功能開發人員已經在 Chrome Web Store 上以他們的名義發布了其他幾個擴充程序，並且所有這些擴充功能都獲得了精選徽章。開發者還在 ExtensionHub 上擁有一個帳戶，儘管目前沒有列出待售的擴充功能。此外，該人還試圖以 2,500 美元的價格出售「AIInfraStack[.]com」等域名，並表示「強關鍵字域名」與[原文如此]快速增長的人工智慧生態系統相關」。

「簡而言之，這就是延伸供應鏈問題，」Annex Security 說。 “經過審查的‘精選’功能擴展易手，新所有者向每個現有用戶推送武器化更新。”

這次披露之際，微軟警告稱，基於 Chromium 的惡意瀏覽器擴充功能會偽裝成合法的 AI 助理工具，以獲取 LLM 聊天歷史和瀏覽資料。

Microsoft Defender 安全研究團隊表示：“從規模上看，這項活動將看似可信的生產力擴展轉變為嵌入日常企業瀏覽器使用中的持久數據收集機制，凸顯了瀏覽器擴展在企業環境中帶來的日益增長的風險。”

最近幾週，威脅追蹤者還標記了一個名為 lmΤoken Chromophore（ID：bbhaganppipihlhjgaaeeeefbaoihcgi）的惡意 Chrome 擴充程序，該擴充功能冒充 imToken，同時在 Chrome 線上應用程式商店中將自己宣傳為十六進位顏色視覺化工具，以利用網路釣魚短語來竊取加密貨幣種子。

Socket 研究員 Kirill Boychenko 表示：“該擴展程序並沒有提供其所承諾的無害工具，而是在安裝後立即自動打開一個由威脅行為者控制的網絡釣魚站點，並且在用戶單擊它時再次打開。”

「安裝後，該擴充功能會從硬編碼的 JSONKeeper 端點 (jsonkeeper[.]com/b/KUWNE) 取得目標 URL，並開啟一個指向類似 Chrome Web Store 風格的網域 chroomewedbstorre-detail-extension[.]com 的標籤。登陸頁面使用混合腳本同形文字模擬 imToken1字的憑證捕獲流中。

Palo Alto Networks Networks Unit 42 標記的其他惡意擴展被發現參與附屬劫持和資料洩露，其中之一 - Chrome MCP 伺服器 - AI 瀏覽器控制（ID：fpeabamapgecnidibdmjoepaiehokgda） - 充當成熟的遠端存取木馬，同時使用模型上下文協議 (MCP) 偽裝成 AI 自動化工具。

Unit 42 研究人員還透露，三個流行的 Chrome 擴充功能——Urban VPN Proxy、Urban Browser Guard 和 Urban Ad Blocker——在之前因從各種聊天機器人（包括 OpenAI ChatGPT、Anthropic Claude、Microsoft Copilot、DeepSeek、Google Gemini、xAI Grok、Meta Store Web 和 Perplex 對話而被刪除後被刪除，

研究人員 Qinge Xie、Nabeel Mohamed、Shresta Bellary Seetharam、Fang Liu、Billy Melicher 和 Alex Starov 表示：「在 2025 年 12 月 15 日公開披露該活動後，開發人員於 2026 年 1 月更新了良性版本，這可能是對報告的回應。」

此外，該網路安全公司還發現了一個名為 Palette Creator（ID：iofmialeiddolmdlkbheakaefefkjokp）的擴展，該擴展擁有超過 100,000 名用戶，其先前版本與與 RedDirection 活動相關的已知網路指標進行通信，以執行瀏覽器劫持。

這還不是全部。一個包含 30,000 多個網域的新活動被發現啟動重新導向鏈，將流量路由到登陸頁面（「ansiblealgorithm[.]com」），該登陸頁面用於分發名為 OmniBar AI 聊天和搜尋的 Chrome 擴充功能（ID：ajfanjhcdgaohcbphpaceglgpgaaohod）。

該擴充功能利用 chrome_settings_overrides API 來更改 Chrome 設定並將瀏覽器主頁設為omnibar[.]ai，並將預設搜尋提供者設定為自訂 URL：「go.omnibar[.]ai/?api=omni&sub1=omnibar.ai&q={searchTerms}」並透過 API 參數追蹤。
