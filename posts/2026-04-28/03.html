03. 2026-04-28｜微軟 GitHub 在成本危機中轉向計量 AI 計費

全文翻譯（繁體中文・香港）

微軟將關閉向 GitHub Copilot 客戶提供的人工智慧自助餐，承認它無法像 Red Lobster 的 Endless Shrimp 那樣銷售人工智慧。

這家美國海鮮餐廳的蝦子無限量促銷導致該公司在 2024 年破產，儘管微軟在財務上遠沒有那麼過度擴張，但這家軟體巨頭的代碼託管業務已決定不再希望 Copilot 虧損運營。

因此，GitHub 將於 2026 年 6 月 1 日將 Copilot 從基於請求的計費轉變為基於使用的計費。

GitHub 吸收了大部分不斷上升的推理成本，但目前的溢價請求模型不再可持續

GitHub 產品團隊首席產品官 Mario Rodriguez 在一篇部落格文章中解釋道：“如今，一個快速聊天問題和一個長達數小時的自主編碼會話可能會讓用戶付出同樣的代價。” “GitHub 已經吸收了這種使用背後不斷上升的推理成本，但當前的高級請求模型不再可持續。”

在基於使用的計費下，與計量代幣有更直接的關聯——三到四個字元的集合，代表銷售人工智慧服務的基本經濟單位。

這並不像 X 美元換 X 代幣那麼簡單——不同的模型以不同的價格計量代幣——因此 GitHub 設計了一種名為 GitHub AI Credits 的虛擬貨幣單位，價值 0.01 美元。

Copilot 客戶消耗輸入令牌、輸出令牌和快取令牌，每個令牌均根據所使用的模型定價。微軟將其轉換為以 AI 積分衡量的成本。

「每個 Copilot 計劃都將包括每月分配的 GitHub AI Credits，而不是計算高級請求，並且可以選擇付費計劃來購買額外的使用量，」Rodriguez 說。 “使用量將根據令牌消耗量進行計算，包括輸入、輸出和快取令牌，並使用每個模型列出的 API 費率。”

提前知道此計算的結果將很困難 - 基於使用的計費是不確定的，因此用戶永遠無法確定模型將消耗多少時間和多少令牌來回應特定輸入。不同的提示可能涉及使代幣消耗計算複雜化的工具。

GitHub 至少打算嘗試向客戶暗示即將發生的事情。羅德里格斯表示，該公司將在「5 月初推出預覽帳單體驗，讓用戶和管理員在 6 月 1 日過渡之前了解預計成本」。

當 OpenClaw 在 2 月引起廣泛關注時，人工智慧公司感到驚訝，促使人工智慧代理在各種任務上 24/7 運行的實驗激增。而此時人工智慧模型能力的不斷增強也鼓勵了更多開發者探索人工智慧編碼。

因此，透過訂閱計畫提供人工智慧服務補貼的公司面臨著超越推理基礎設施所能滿足的需求。隨後的價格調整已波及整個產業。

GitHub 上週表示，打算透過暫停建立新的 Copilot、Pro、Pro+ 和 Student 計畫來止住虧損。

在此之前，Anthropic 和谷歌採取措施限制其服務的某些使用。 OpenAI 的回應是推出了更昂貴的 100 美元訂閱套餐，以提高其 Codex 模型的使用率，儘管該公司正在考慮結束訂閱計劃下的無限使用。 AWS 和 Azure 等雲端供應商也一直在應對容量挑戰。

GitHub 的訂閱費率將保持不變：Copilot Pro 為 10 美元/月，Pro+ 為 39 美元/月，Business 為 19 美元/用戶/月，Enterprise 為 39 美元/用戶/月。
