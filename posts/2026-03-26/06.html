06. 2026-03-26｜GitHub 按了 CTRL-Z，決定最終將使用使用者資料訓練 AI

全文翻譯（繁體中文・香港）

微軟的 GitHub 下個月計畫開始使用客戶互動資料——「特別是輸入、輸出、程式碼片段和相關上下文」——來訓練其人工智慧模型。

自 4 月 24 日起，代碼鎖修改後的政策適用於 Copilot Free、Pro 和 Pro+ 客戶。 Copilot Business 和 Copilot Enterprise 用戶可依合約條款豁免。使用 Copilot 的學生和教師也將倖免。

受影響的人可以選擇根據「既定的行業慣例」選擇退出——這意味著根據美國規範，而不是通常要求選擇加入的歐洲規範。若要選擇退出，GitHub 使用者應造訪 /settings/copilot/features 並在隱私標題下停用「允許 GitHub 使用我的資料進行 AI 模型訓練」。

GitHub 首席產品官 Mario Rodriguez 寧願你不這樣做。

他在部落格文章中寫道：“透過參與，您將幫助我們的模型更好地理解開發工作流程，提供更準確、更安全的程式碼模式建議，並提高它們在生產之前幫助您捕獲潛在錯誤的能力。”

為了為其貪婪的行為開脫，GitHub 在其常見問題中指出，Anthropic、JetBrains 和母公司 Microsoft 也實施類似的選擇退出資料使用政策。

羅德里格斯表示，這項變化的理由是互動數據使公司人工智慧模型表現更好。他聲稱，添加微軟員工的互動數據帶來了有意義的改進，例如人工智慧模型建議的接受率提高。

這項政策轉變確實在一定程度上改變了 GitHub 私有儲存庫的含義，理論上「只有您、您明確與之共享存取權限的人才能訪問，對於組織儲存庫，只有某些組織成員才能存取」。這些可能更準確地描述為“GitHub 私有 * 儲存庫”，星號表示 GitHub 對“私有”一詞定義的限制。

正如常見問題解答所解釋的那樣：“如果 Copilot 用戶的設定設置為對其交互數據啟用模型訓練，則可以收集私有存儲庫中的代碼片段並將其用於模型訓練，同時用戶在該存儲庫中工作時積極與 Copilot 互動。”

GitHub 社群最近的玩笑並沒有體現出對該計劃的太多熱情。僅從表情符號投票來判斷，用戶投了 59 票反對票，只有三艘火箭飛船，我們知道這表明了某種程度的興奮。

但在撰寫本文時評論這項變更的 39 篇文章中，除了 GitHub 開發者關係副總裁 Martin Woodward 之外，沒有人真正支持這個想法。

如果 GitHub 使用者認識到 GitHub Copilot 中使用的 OpenAI 的 Codex 是“在 GitHub 公開可用的程式碼上進行微調的 GPT 語言模型”，那麼使用者的憤怒可能會有所緩解。可以這麼說，這些措辭表明，充滿數據的人工智慧馬已經走出了穀倉。

此時關上大門並不會改變這樣一個事實：人工智慧產業是建立在收集數據的基礎上，而不需要強烈的熱情同意指標。 ®
