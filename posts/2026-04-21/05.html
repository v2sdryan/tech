05. 2026-04-20｜微軟 GitHub 因容量緊張而暫停 Copilot 帳戶註冊

全文翻譯（繁體中文・香港）

微軟的 GitHub 已停止接受新的 Copilot 個人訂閱，而程式碼託管公司正在研究如何在不花太多錢的情況下履行其服務承諾。

產品副總裁 Joe Binder 在周一的部落格文章中寫道，程式碼鎖已暫停 GitHub Copilot Pro、Pro+ 和 Student 計畫的註冊，以幫助公司更有效地服務現有客戶。

「代理商工作流程從根本上改變了 Copilot 的運算需求，」Binder 說。 「現在，長時間運行的並行會話通常消耗的資源遠多於最初建立的計劃結構所支援的資源。

“隨著 Copilot 的代理能力迅速擴展，代理商正在做更多的工作，更多的客戶達到了旨在維持服務可靠性的使用限制。如果不採取進一步行動，每個人的服務品質都會下降。”

微軟沒有說明為什麼需要暫停，但 2 月對 OpenClaw 的熱情高漲似乎讓人工智慧基礎設施供應商對不斷增長的需求毫無準備，難以跟上。

Anthropic 試圖透過調整其使用限制以將消費轉移到高峰時段並強制執行有關使用 OpenClaw 等第三方工具的政策來減少需求。谷歌為其反重力人工智慧開發環境、Gemini CLI 和 Gemini Code Assist 制定了類似的政策。 OpenAI 本月稍早進行了自己的使用平衡。

雲端供應商也很難跟上腳步。據報道，AWS 去年因無法滿足人工智慧需求而輸給了谷歌雲端。微軟Azure最近也遇到了容量問題。 GitHub 也一直在為可用性而苦苦掙扎。

去年，科技業一直在談論軟體代理。但支援自主軟體和人工智慧工作負載激增的基礎設施仍在進行中，一些正在建造的用於處理負載的資料中心的工作已經停止、放緩或放棄。現在，隨著 Anthropic 和 OpenAI 尋求上市，領先的模型製造商面臨著縮小損失的壓力，這使得昂貴的資料中心建設變得不再那麼令人胃口。

對於 GitHub 來說，計算短缺的最新表現是上週因濫用而暫停了 GitHub Copilot Pro 免費試用。 GitHub Copilot 的免費套餐仍然可用。

現在，作為 GitHub 成本削減和服務調整的一部分，Binder 表示，該行動將收緊個人計劃的使用限制。

GitHub Copilot 實施兩種形式的使用限制：會話限制和每週限制，它們與令牌消耗和特定於模型的乘數相關。

Binder 表示，會話限制有助於確保模型在高峰使用期間保持可用。他說，GitHub 將調整這些「以平衡可靠性和需求」。超過會話限制意味著要等到使用視窗重置後才能再次使用 Copilot。

每週限制限制了用戶在一周內可以消耗的代幣數量。 Binder 表示，引入它們是為了“控制並行的、長軌蹟的請求，這些請求通常會運行很長時間並導致成本過高。”

本月早些時候推出了一套單獨的高級使用限制，限制了對高端型號的請求。

GitHub Copilot 目前按請求計費，這是用戶與 Copilot 進行的任何交互，而不是按令牌計費。如果請求最終使後端模型陷入意想不到的長思路，那麼透過模型乘數修改的統一費率最終仍可能使微軟付出比公司收費更高的成本。

因此，據報道，這些最新的變化反映了朝著基於代幣計費的方向努力，並放棄提供統一費率代幣消費的計劃。
