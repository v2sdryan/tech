04. 2026-04-22｜Anthropic 測試開發人員對從 Pro 計劃中刪除 Claude Code 的反應

全文翻譯（繁體中文・香港）

根據一些面向公眾的網頁顯示，Anthropic 已將 Claude Code 從其 Pro 訂閱計劃中刪除，但該公司表示這只是針對少數用戶的測試。

週一，該公司的定價網頁稱其 Pro 計劃“包括 Claude Code”。週二，該短語不再出現，並且在頁面的其他位置，Pro 計劃的功能列表包含一個明確的“X”，而不是 Claude Code 的複選標記。人工智慧產業懷疑論者 Ed Zitron 先前曾指出這些變化。

Anthropic 更改的產品內容尚未覆蓋其網站的所有角落。在撰寫本文時，Claude Code 產品頁面仍然指出 Pro 計劃提供對程式碼產生工具的存取。當記者從CLI訪問Claude Code時，終端輸出仍然顯示「Claude Pro」。

顯示 Claude Code 訂閱的不同 Anthropic 網頁的螢幕截圖 - 點擊放大

Claude.ai 在被問及此事時堅稱 Pro 計劃包括 Claude Code，但今天更新的引用文檔頁面之一隻提到了 Max 計劃。

失去對 Claude Code 的訪問權讓開發人員感到震驚，這種情緒也傳到了 Anthropic，其增長主管 Amol Avasare 發表了一篇社交媒體帖子，表明這一變化代表了一項實驗。

為什麼 Anthropic 認為其面向公眾的定價頁面的更新只會針對部分用戶而不是整個互聯網，這一點尚不完全清楚。公司發言人沒有回應有關此事的後續問題。

Avasare 繼續解釋道：「當我們一年前推出 Max 時，它不包含 Claude Code，Cowork 也不存在，運行幾個小時的代理也不是問題。Max 是為大量聊天使用而設計的，僅此而已。

「從那時起，我們將 Claude Code 捆綁到 Max 中，並在 Opus 4 後開始流行。Cowork 落地。長時間運行的非同步代理現在已成為日常工作流程。人們實際使用 Claude 訂閱的方式已經從根本上改變了。

“因此，我們正在尋找不同的選項，以繼續為用戶提供出色的體驗。我們還不知道這些選項到底是什麼樣的——這就是我們現在正在測試並獲取反饋的內容。”

Anthropic 最近一直在採取措施限制其人工智慧模式的使用，以抑制產能緊張的需求，這也是 GitHub 和谷歌面臨的問題。各種形式的 Claude 在過去幾個月中變得非常流行，OpenClaw 等專案的出現導致許多開發人員使用 Claude 模型建立代理程式。

基本問題是 Anthropic 的訂閱方案收費遠低於所消耗代幣的帳面價值，有時甚至低十倍或更多。該公司上個月引入了使用限制，以鼓勵在非高峰時段使用，就像電力公司為促進節能而採取的措施一樣。正如 Avasare 所指出的，Anthropic 正在探索進一步的使用限制。

這種轉變的引人注目之處不在於它的發生，而在於缺乏與客戶之間的連貫溝通。也許 Anthropic 認為不需要向每月僅支付 20 美元的個人客戶提供公平的警告或明確的計劃變更公告。

然而，企業客戶不喜歡這種不確定性，並可能擔心 Anthropic 不將客戶關係視為優先事項。
