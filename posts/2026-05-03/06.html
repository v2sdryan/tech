全文翻譯（繁體中文・香港）

隨著模型開發人員推行更激進的速率限制、提高價格，甚至放棄基於使用的定價的訂閱，這種振動編碼的愛好項目將變得更加昂貴。幸運的是，您並非沒有節省成本的選擇。

在過去的幾周里，我們看到 Anthropic 玩具從其最實惠的計劃中刪除了 Claude Code，而微軟則跳過了試水，並將 GitHub Copilot 轉向純粹基於使用的模型。整個災難讓我們思考。我們是否需要 Anthropic 或 OpenAI 的頂級模型，或者我們可以使用較小的本地模型嗎？當然，它可能會更慢，能力更差，並且使用起來更令人沮喪，但你無法擊敗免費的價格......好吧，假設你已經擁有了硬體。

碰巧阿里巴巴最近放棄了 Qwen3.6-27B，這家雲端和電子商務巨頭聲稱將「旗艦編碼能力」打包到一個足夠小的包中，可以在 32 GB M 系列 Mac 或 24 GB GPU 上運行。

這不是我們第一次研究本地代碼助手。之前我們曾探索使用Continue 的VS Code 擴充功能來完成程式碼完成和產生等任務。

當時，模型和軟體堆疊還相當不成熟，這使得它們成為有用的工具，但不一定足以與更大的前緣模型競爭。從那時起，模型架構和代理工具得到了顯著改進。

「推理」功能允許小型模型透過「思考」更長的模型來彌補其尺寸，混合專家模型意味著您不需要每秒 TB 級的記憶體頻寬來獲得互動體驗，而大大改進的函數和工具呼叫功能意味著這些模型實際上可以與程式碼庫、shell 環境和 Web 進行互動。

在本次實踐中，我們將了解如何部署和配置 Qwen3.6-27B 等本地模型，以便在電腦上進行編碼，並探索一些可以與它們一起使用的代理框架。

注意：較舊的 M 系列 Mac 可能難以應對代理編碼所需的大上下文長度。您可能會更幸運地使用像 oMLX 這樣的推理引擎，它可以更好地利用 Apple 的硬體加速器，但您的情況可能會有所不同。

如今，在本地運行法學碩士是一個非常簡單的過程。安裝您最喜歡的推理引擎。下載模型，並透過 API 連接您的應用程式。
