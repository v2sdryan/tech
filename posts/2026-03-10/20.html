20. 2026-03-09｜UNC4899 透過 AirDrop 木馬檔案入侵加密貨幣公司

全文翻譯（繁體中文・香港）

被稱為 UNC4899 的北韓威脅行為者被懷疑是 2025 年針對加密貨幣組織的複雜雲端妥協活動的幕後黑手，目的是竊取數百萬美元的加密貨幣。

該活動被認為是由國家資助的對手所為，並以密碼 Jade Sleet、PUKCHONG、Slow Pisces 和 TraderTraitor 進行追蹤。

這家科技巨頭在與《駭客新聞》分享的 2026 年上半年雲威脅視野報告中指出，“這一事件因其融合了社會工程、利用個人到企業設備點對點數據 (P2P) 傳輸機制、工作流程以及最終轉向雲以採用離雲 (LOTC) 技術而引人注目。”

據稱，在獲得雲端環境存取權限後，攻擊者濫用合法的 DevOps 工作流程來取得憑證、突破容器的限制並篡改 Cloud SQL 資料庫，以促進加密貨幣盜竊。

谷歌雲端表示，攻擊鏈代表了從開發人員的個人裝置入侵其公司工作站開始，然後跳到雲端對財務邏輯進行未經授權的修改的過程。

這一切都始於威脅行為者使用社會工程策略欺騙開發人員下載存檔文件，作為所謂的開源專案協作的一部分。然後，開發人員透過 AirDrop 將相同的文件傳輸到他們的公司設備。

谷歌表示：“受害者使用人工智慧輔助的整合開發環境（IDE）與存檔內容進行交互，最終執行嵌入的惡意 Python 程式碼，該程式碼生成並執行了一個偽裝成 Kubernetes 命令行工具的二進位檔案。”

然後，該二進位檔案會聯繫攻擊者控制的網域，並充當受害者公司電腦的後門，為攻擊者提供了一種透過可能使用經過驗證的會話和可用憑證轉向 Google Cloud 環境的方法。此步驟之後是初始偵察階段，旨在收集有關各種服務和項目的資訊。

隨著發現堡壘主機，攻擊進入下一階段，攻擊者修改其多重身份驗證 (MFA) 策略屬性以存取它並執行額外的偵察，包括導航到 Kubernetes 環境中的特定 Pod。

隨後，UNC4899採用了一種脫離雲端（LotC）的方法，透過改變Kubernetes部署配置來設定持久化機制，以便在建立新pod時自動執行bash指令。該命令本身下載了一個後門。

谷歌表示，這起事件「凸顯了個人與企業的 P2P 資料傳輸方法和其他資料橋、特權容器模式以及雲端環境中不安全的秘密處理所帶來的嚴重風險」。 「組織應該採用深度防禦策略，嚴格驗證身份，限制端點上的資料傳輸，並在雲端運行時環境中強制實施嚴格隔離，以限制入侵事件的影響範圍。”

為了應對威脅，建議組織實施上下文感知存取和防網釣魚 MFA，確保僅部署受信任的映像，隔離受感染的節點，使其無法與外部主機建立連接，監視意外的容器進程，採用強大的機密管理，強制執行策略以禁用或限制使用 AirDrop 或藍牙的點對點文件共享，以及在企業設備上安裝非託管外部媒體。

與受害者的 CI/CD 平台解決方案相關的 Kubernetes 資源被修改為注入在日誌中顯示服務帳戶令牌的命令。

攻擊者獲得了高權限 CI/CD 服務帳戶的令牌，允許他們升級權限並進行橫向移動，特別針對處理網路策略和負載平衡的 Pod。

被盜的服務帳戶令牌用於對以特權模式運行的敏感基礎設施 Pod 進行身份驗證、逃離容器並部署後門以進行持久存取。

威脅行為者進行了另一輪偵察，然後將注意力轉移到負責管理客戶資訊的工作負載上，例如用戶身份、帳戶安全性和加密貨幣錢包資訊。

攻擊者用它來提取不安全地儲存在 Pod 環境變數中的靜態資料庫憑證。

然後，這些憑證被濫用，透過 Cloud SQL Auth 代理程式存取生產資料庫並執行 SQL 命令來修改使用者帳戶。其中包括多個高價值帳戶的密碼重設和 MFA 種子更新。

這次攻擊最終以利用受損帳戶成功提取數百萬美元的數位資產而告終。
