11. 2026-04-23｜開發者抱怨 Claude Opus 4.7 變成了一個過於熱心的查詢警察

全文翻譯（繁體中文・香港）

Anthropic 上週發布的 Opus 4.7 附帶了更強有力的保護措施來防止濫用。不幸的是，這些保護措施也成功地阻礙了合法使用。

Opus 4.7 是在 Anthropic 宣布推出 Mythos 之後發布的，據稱該模型太有能力發現和利用漏洞，無法向公眾公開。雖然這似乎是對該模型風險的自私評估，但該公司決定使用 Opus 4.7 作為高度警惕護欄的測試平台。

「我們正在發布帶有保護措施的 Opus 4.7，可以自動檢測和阻止表明禁止或高風險網路安全用途的請求，」人工智慧公司表示。 “我們從現實世界部署這些保障措施中學到的經驗將幫助我們實現廣泛發布 Mythos 級模型的最終目標。”

Anthropic 可以透過仔細研究 Claude Code 的 GitHub 儲存庫中的投訴來學到很多東西。對公司可接受使用政策 (AUP) 分類器的反對意見激增，客戶在完成合法工作時遇到了困難。

安全性越高，誤報就越多——克勞德變得過於謹慎，拒絕回應無害的請求。克勞德 (Claude) 編制的關於 AUP 拒絕的投訴圖表講述了這個故事：

克勞德關於模型拒絕問題的圖表 - 點擊放大

幾個月來，Claude Code 用戶一直對 GitHub 問題貼文中的無效拒絕表示擔憂，但直到最近，這一比例一直相當穩定。

從 2025 年 7 月到 2025 年 9 月，每月大約有兩到三起此類投訴。其中一個問題是#4373，「claude.ai 的記憶體授權程式碼觸發 API 策略錯誤」。

從 2025 年 10 月到 2025 年 11 月，與 AUP 相關的拒絕增加到每月約 5 到 7 次，其中包括 #8784 、「Claude 4.5 拋出 API 錯誤：Claude Code 無法隨機回應此正常請求」等問題。

12月份，相關投訴較少，可能是美國假期放緩所致。

一月份，投訴數量回到了八個左右。提交#16129「Claude 程式碼中重複出現虛假 AUP 違規」的開發人員表示，「技術軟體開發對話不應觸發 AUP 違規。安全過濾器對良性內容顯得過於激進。」二月和三月的數字相似。

4 月份，開發人員已提交 30 多份報告，聲稱存在與安全、一般開發用途和科學拒絕相關的誤報。

最近還有許多其他可疑拒絕的例子，包括： #50795 、 #51352 、 #51794 、 #52086 、 #50494 、 #49904 、 #46147 和 #51248 。

部分成長可能歸因於用戶群的不斷增長。克勞德客戶越多，報告問題的人就越多。但顯然也有很多 Claude 用戶被過於激進的 AUP 分類器關閉。

鑑於洩漏的 Claude 程式碼來源如何使用正規表示式模式進行情緒分析，AUP 分類器可能會採取類似的捷徑，僅檢查禁用字詞而不考慮上下文。

Anthropic 沒有回應置評請求。 ®
