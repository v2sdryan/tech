11. 2026-04-17｜CISA 要求聯邦政府修補受到主動攻擊的 13 年之久的 Apache ActiveMQ 漏洞

全文翻譯（繁體中文・香港）

CISA 對新利用的 Apache ActiveMQ 漏洞發出警報，要求聯邦機構在兩週內進行修補，因為攻擊者已經發現了一個已經悄悄潛伏了十多年的漏洞。

美國網路安全機構週四將這個編號為 CVE-2026-34197 的漏洞添加到其已知可利用漏洞 (KEV) 目錄中，從而觸發了具有約束力的操作指令 (BOD) 22-01 的最後期限，該指令要求聯邦民事行政部門機構在 4 月 30 日之前修復其係統或準備好解釋原因。

該漏洞存在於 Apache ActiveMQ 中，Apache ActiveMQ 是一個開源訊息代理，用於在應用程式和服務之間傳輸數據，並允許經過身份驗證的使用者透過代理的 Jolokia 管理 API 執行任意程式碼，從而有效地將訊息傳遞主力轉變為遠端命令執行程式。

就在一周前，Horizo​​​​n3 研究員 Naveen Sunkavally 披露了這一消息，他使用 Anthropic 的 Claude AI 助手來幫助挖掘它。據 Horizo​​n3 稱，這個問題已經在程式碼庫中存在了 13 年，直到現在才被注意到。 ActiveMQ 版本 5.19.5 和 6.2.3 中提供了補丁。

Sunkavally 表示：「CVE-2026-34197 是 Apache ActiveMQ Classic 中的一個遠端程式碼執行漏洞，它已經隱藏在眾目睽睽之下 13 年了。」「攻擊者可以透過 ActiveMQ 的 Jolokia API 呼叫管理操作，以欺騙代理程式取得遠端設定檔並執行任意作業系統命令。」

雖然該錯誤從技術上講需要身份驗證，但 Horizo​​n3 指出，許多部署仍然依賴預設憑證 - 始終可靠的「admin:admin」 - 使得初始存取變得微不足道。更糟的是，在某些版本（6.0.0 到 6.1.1）上，較舊的缺陷 CVE-2024-32114 可以在完全沒有身份驗證的情況下暴露 Jolokia API，從而將其轉變為無需憑證的遠端程式碼執行鏈。

Sunkavally 說：“漏洞需要憑證，但預設憑證在許多環境中很常見。” “在某些版本上......根本不需要憑證......在這些版本中，CVE-2026-34197 實際上是未經身份驗證的 RCE。”

這種組合正是在 CISA 的 KEV 清單中出現錯誤的類型，該清單是為已經在野外利用的漏洞保留的。還有大量的暴露表面可供瞄準：威脅監控機構 ShadowServer 正在追蹤可從公共互聯網存取的 8,000 多個 ActiveMQ 實例。

這也不是 ActiveMQ 第一次與攻擊者發生衝突。該平台的特點是存在相當多的妥協，從加密貨幣礦工到殭屍網路基礎設施。正如 Sunkavally 指出的那樣，這一切都不是特別新穎，這讓管理員有責任迅速採取行動。 ®
