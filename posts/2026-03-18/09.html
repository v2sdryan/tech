09. Tue, 17 Ma｜為什麼 Garry Tan 的 Claude Code 設定讓人又愛又恨

全文翻譯（繁體中文・香港）

Y Combinator 著名的執行長 Garry Tan 告訴 SXSW 觀眾，他患有“網路精神病”，幾乎無法入睡，因為他對與人工智慧代理商合作感到非常興奮。

「我現在每晚睡四個小時，」他在周六的台上採訪中告訴他的採訪者、風投同事 Bill Gurley。 「我有網路精神病，但我認為我認識的執行長中有三分之一也有這種病，」他開玩笑地談到自己目前對人工智慧的痴迷。 （至少，我們希望他是在開玩笑。人工智慧引起的精神病實際上可能是一種危險的情況。）

「一旦你嘗試過，你就會意識到：這就像我能夠重新創建我的初創公司，花了 1000 萬美元的風險投資和 10 個人，我為此工作了兩年，我服用了抗嗜睡藥 - 我記得，你知道，有點像服用莫達非尼，」他描述道，指的是在創業熱潮中流行的一種防止睡眠的藥物人群中。 （Tan 早在 2012 年就將他由 Y Combinator 支持的部落格新創公司 Posterous 賣給了 Twitter。）

但現在，他的精神因與人工智慧代理一起工作而變得如此興奮，這是一種自然的失眠。

「這場革命不需要莫達非尼。就像，我起床了。我凌晨 4 點睡覺，早上 8 點醒來，」他說。 “我想多睡一會兒，但睡不著，因為：讓我們看看 10 名工人的情況如何。我現在有三個不同的項目正在進行。”

他對他的代理感到非常興奮，以至於 3 月 12 日，就在面試前兩天，他自豪地在 GitHub 上根據開源許可免費分享了他的 Claude Code (CC) 設定。該設定包括他開發的六種「固執己見」的克勞德代碼技能。技能是儲存在特殊「skill.md」檔案中的可重複使用的提示，指示 AI 如何在特定角色或任務中表現。

「我和 Claude Code 一起度過了一段美妙的時光，我希望你能夠擁有我的*精確的*技能設置，」他在 X 上發文說。他將他的 Claude Code 設定稱為“gstack”。

gstack 現已在 https://t.co/VPvWDzV5c0 提供 開源、MIT 許可證，請告訴我它是否適合您。只需貼上一次即可將其安裝到您本地的 Claude Code 上，而將其安裝到您的隊友的儲存庫中則是第二次貼上。

從那時起，他又增加了一些技能。 gstack GitHub 儲存庫目前列出了 13 個，但 Tan 似乎每小時都會發布一些新的推文。

在一篇文章中，他舉了一個例子來說明他的設定是如何運作的。首先，他使用克勞德充當執行長的技能，以了解克勞德對創業想法或功能是否是一個好主意的看法。他使用另一項技能讓克勞德作為工程師編寫功能，並使用另一項技能作為程式碼審查員來審查自己的工作是否有錯誤和安全問題。其他技能包括設計、文件等。

對 gstack 的熱愛立即開始：他的推文在 X 上瘋傳，並在 Product Hunt 上成為熱門話題。它在 GitHub 上累積了近 20,000 顆星，有 2,200 個“分叉”，這意味著人們拿走了文件並自行修改。

但在發布 gstack 後不久，Tan 發布了一條推文，也引起了一堆仇恨。

他寫道，一位 CTO 朋友告訴他 gstack 是“上帝模式”，立即發現了他公司代碼中的安全漏洞，並預測它將被廣泛使用。

我的 CTO 朋友發短信給我：“你的 gstack 太瘋狂了。這就像上帝模式。你的工程師審查發現了一種微妙的跨站點腳本攻擊，我什至認為我的團隊沒有意識到這一點。我敢打賭，從今天開始，超過 90% 的新存儲庫將使用 gstack。” https://t.Fuco/P7aO55

引用隨後眾多仇恨評論中的一小部分：一位創始人在 X 上發帖稱：“(1) Garry 應該為發推文感到尷尬。(2) 如果屬實，該 CTO 應該立即被解僱。”

Vlogger Mo Bitar 對 gstack 做了一個題為「人工智慧讓 CEO 產生妄想」的文章，他指出該專案本質上是文字檔案中的「一堆提示」。這位影片部落客總結了常見的抱怨：使用 Claude Code 的開發人員已經有了自己的版本。
