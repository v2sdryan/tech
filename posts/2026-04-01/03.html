03. Tue, 31 Ma｜人類正在度過一個月

全文翻譯（繁體中文・香港）

Anthropic 圍繞著其作為一家謹慎的人工智慧公司的成功理念建立了自己的公眾形象。它發表了有關人工智慧風險的詳細工作，聘請了該領域一些最優秀的研究人員，並且一直直言不諱地表達了構建如此強大的技術所帶來的責任——當然，它現在正在與國防部展開激烈的鬥爭。可惜的是，週二，那裡有人忘記勾選一個複選框。

值得注意的是，這是一週內的第二次。上週四，《財星》雜誌報道稱，Anthropic 無意中公開了近 3,000 份內部文件，其中包括一篇部落格文章草稿，描述了該公司尚未宣布的強大新模型。

這是周二發生的事情：當 Anthropic 推出其 Claude Code 軟體包 2.1.88 版本時，它意外地包含了一個文件，該文件暴露了近 2,000 個源代碼文件和超過 512,000 行代碼——本質上是其最重要產品之一的完整架構藍圖。一位名叫 Chaofan Shou 的安全研究員幾乎立即註意到了這一點，並在 X 上發布了相關資訊。 Anthropic 對多家媒體的聲音明顯得漠不關心：「這是人為錯誤導致的發布包裝問題，而不是安全漏洞。」(在內部，我們猜測事情的衡量程度較低。)

Claude Code 並不是一個小產品。它是一個命令列工具，允許開發人員使用 Anthropic 的人工智慧來編寫和編輯程式碼，並且已經變得強大到足以讓競爭對手感到不安。根據《華爾街日報》報道，OpenAI 在向公眾推出視訊生成產品 Sora 僅僅六個月後就停止了該產品，將其工作重點重新集中在開發人員和企業上，部分原因是為了應對 Claude Code 的成長勢頭。

洩漏的不是人工智慧模型本身，而是圍繞它的軟體支架——告訴模型如何行為、使用什麼工具以及其限制的指令。開發人員幾乎立即開始發布詳細的分析，其中有人將該產品描述為「一種生產級的開發人員體驗，而不僅僅是 API 的包裝器」。

這是否會產生持久的影響，這個問題最好留給開發人員。競爭對手可能會發現該架構具有指導意義；同時，這個領域也在快速發展。

不管怎樣，在 Anthropic 的某個地方，你可以想像一位非常有才華的工程師花了一天剩下的時間靜靜地想知道他們是否還有工作。人們只能希望這不是上週晚些時候的同一個工程師或工程團隊。
