15. 2026-05-05｜Anthropic希望克勞德玩錢，釋放財務代理

全文翻譯（繁體中文・香港）

如果您曾經讀過 Anthropic 的免責聲明，即克勞德產生的回應可能包含錯誤，並認為“這就是我需要加強財務運營的內容”，那麼您很幸運。

Anthropic發布了一套財務代理模板，旨在讓其Claude AI服務更好地協助財務任務。

該公司解釋道：“每個代理模板都是一個參考架構，其中包含三項內容：技能（任務的指令和領域知識）、連接器（對任務運行數據的受控訪問）和子代理（主代理調用的附加克勞德模型，用於特定的子任務，例如可比較選擇或方法檢查）。”

這個術語可能有點模糊，因為歸根結底，這只是一個使用工具和資料等資源在迭代循環中追求目標的模型。

Claude Code 本身就是一個代理工具，支援使用 Anthropic 定義的控制流的底層模型。當克勞德模型推動控制流實現目標時——決定使用哪些工具以及存取哪些資料——這就是代理。

然後是子代理，這些實際上只是使用專門的系統提示、指定的工具和編排系統提供的上下文對 Claude 進行 API 呼叫。它們有點像程式中處理應用程式特定方面的函數。

因此，Anthropic 的財務代理包括： 技能，是描述工作流程的 Markdown 文件；連接器，與外部服務整合；和子代理，由重點系統提示、特定工具和上下文資料組成。

例如，Anthropic 的「了解您的客戶篩選器」代理程式範本 (kyc-screener) 包含一項名為 kyc-rules 的技能，該技能詳​​細說明了 Claude 應如何將公司的 KYC/AML（反洗錢）規則應用於已解析的入職記錄。這些規則告訴人工智慧模型分配風險評級、檢查文件、引用規則結果並產生格式如下的結果：

Anthropic 的代理名單包括： 推廣搭建者；會議準備者；收益審核員；模型建構者；市場研究員；估價審核員；總帳調節器；月底臨近；報表審核員；以及，如前所述，KYC 篩選器。

這些可以作為插件或作為 Claude Managed Agents 的「食譜」（可複製的程式碼片段）應用於 Claude Cowork 和 Claude Code。

您可能會認為，當涉及到數字等科學事物時，金融往往相當無情。也許您對 Anthropic 的 Opus 4.7 模型在 Vals AI 的 Finance Agent 基準測試中獲得“行業領先”64.37% 的成績不以為然，這個失敗率足以讓人類感到震驚。

不用擔心，因為 Anthropic 希望用戶“堅定地參與循環——在 Claude 的工作交給客戶、提交或採取行動之前對其進行審查、迭代和批准。”
