10. 2026-04-28｜OpenAI 從微軟的床上跳出來，進入亞馬遜的基岩

全文翻譯（繁體中文・香港）

OpenAI 的頂級模型已在 Amazon Web Services 的 Bedrock 託管推理和代理平台上正式可用。

此次合作於週二在舊金山舉行的 AWS 活動上宣布，為存取 Altman 和公司不斷增長的 GPT 庫提供了另一種途徑，而無需將資料暴露給 OpenAI 的 API。

亞馬遜聲稱，企業希望使用 OpenAI 的模型建立代理和其他人工智慧增強工具，但由於安全政策、資料隱私和主權問題而被阻止。

透過向受信任的第三方開放其模型，OpenAI 可以迴避許多此類問題。將其模型引入 AWS 也意味著客戶無需經歷幾乎同樣多的困難即可採用其模型，因為亞馬遜已經努力將其服務連接到 Bedrock。

除了託管推理服務外，OpenAI 的模型還將在 Amazon 的 Bedrock Managed Agents 和 AgentCore 平台上提供，這些平台提供了用於建立企業代理並將其連接到企業資料和服務的工具和藍圖。 AWS 也同時為其自己的最終客戶推出了一系列新的代理人工智慧工具，包括 Quick（類似於 Microsoft Copilot 但適用於來自多個供應商的應用程式的個人化助手）以及各種新風格的 Connect（最初是亞馬遜託管的 CRM 產品，但正在擴展以幫助客戶自動化人力資源、醫療保健和供應管理中的任務）。

最後，企業將能夠將 OpenAI 的 Codex 程式碼代理程式連接到 AWS 資料中心中運行的模型，從而在一定程度上保證其程式碼庫不會出現在 Altman 的下一個模型中。

目前，在 AWS 上訪問 OpenAI 模型的訪問仍處於有限預覽狀態，LLM 製造商的第二個最新 GPT-5.4 模型現已推出；根據 AWS 首席執行官 Matt Garman 週二在舊金山舉行的一次活動上的講話，後來的 GPT-5.5 將在未來幾週內推出。

週二的公告兌現了 OpenAI 在 2 月的承諾，即在 AWS 上提供其模型，以換取高達 350 億美元的新融資。然而，要獲得這一切，OpenAI 必須啟動 2 千兆瓦的亞馬遜 Trainium 加速器。

這似乎在很大程度上要歸功於微軟願意開放與 OpenAI 的關係，以換取擺脫其收入分享承諾。

根據新條款，微軟仍然是 OpenAI 的主要雲端供應商，並保留對模型開發技術的存取權。同時，OpenAI 可以自由地與任何牠喜歡的人上床，無論是亞馬遜還是其他人。

因此，新條款意味著 OpenAI 與亞馬遜的合作可能不是一次性的，而是未來基礎設施和服務交易的藍圖。 ®
