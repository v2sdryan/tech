09. 2026-04-13｜據克勞德說，克勞德的情況越來越糟

全文翻譯（繁體中文・香港）

Anthropic 的 Claude 曾經是世界各地程式設計師的人工智慧寵兒，但無論是在成本還是感知品質方面都陷入了困境。週一，該服務因「嚴重中斷」而短暫中斷，服務問題只會加劇客戶日益增長的不滿情緒，甚至連機器人都能看到。

這次故障導致錯誤率升高，在世界標準時間 15:31 至 16:19 期間影響了 Claude.ai 和 Claude Code。

26 年 4 月 13 日中斷的螢幕截圖 - 點擊放大

這還不是全部。根據社群媒體貼文以及 GitHub 上提交的問題，在過去的幾個月裡，克勞德的回答越來越不令人滿意。發生這種情況是因為 Anthropic 必須採取措施減少高峰時段的使用量，以平衡容量和需求。

為了獲得更客觀的測量，我們將 Claude 本身指向 Claude Code GitHub 存儲庫，過濾提及質量的開放問題，並給出以下提示：“分析並繪製自 2026 年 1 月以來此存儲庫中有關 Claude Code 質量的投訴。使用提及質量問題的開放問題。這些問題最近是否有所增加？”

Anthropic 的人工智慧模型得出的結論是：“是的，品質投訴急劇升級——數據講述了一個非常清晰的故事。”

Claude 產生的 Claude 程式碼品質問題評估 - 點擊放大

我們要求 Claude 在周一重新進行自我分析，結果相似，模型發出“速度值得注意：4 月份在 13 天內已經出現了 20 多個質量問題，預計將超過 3 月份的 18 個——這本身就比 1 月至 2 月的基線躍升了 3.5 倍。”

克勞德本身並不是一個可靠的敘述者，僅僅因為有人（或某個機器人）向克勞德程式碼儲存庫報告了問題，並不意味著該報告是準確或有效的。現在看來許多問題都是人工智慧產生的——這是開源開發人員廣泛報導的問題——這可能會增加報告數量。

Anthropic 的 GitHub Actions 腳本似乎還會在一段時間不活動後自動關閉問題，這可能有助於掩蓋未解決的問題。

The Register 報告了 Claude 在分析中指出的一些問題，例如快取問題，以及 AMD 人工智慧總監 Stella Laurenzo 聲稱 Claude 的反應變得越來越糟。其他尚未得到證實，例如一項聲稱“克勞德自主刪除了屬於真實付費客戶（JIXEN）的 35,254 條生產客戶消息記錄和 35,874 條計費交易。”

此帖子背後的個人或機器人帳戶沒有發布其他帖子。 The Register 試圖聯繫 Jixen Enterprises Private Limited（該公司似乎是一家在印度註冊的私人公司）來核實這一說法，但我們尚未收到回應。開發人員報告了使用 Claude Code 和其他模型導致的資料遺失。但如果發生這種情況，沒有人排除使用者錯誤的可能性。

無論如何，Claude 能夠引用實際的 GitHub 問題貼文來證明其「推理」的合理性，因此總體趨勢 - 關於品質的報告越來越多 - 是顯而易見的。

該模型指出了諸如“Claude Code 的預測優先行為對於資本風險項目來說是危險的”#46212、“Claude Code 無法用於 2 月更新的複雜工程任務”#42796（由 Claude Code 負責人 Boris Cherny 解決）、“人工降級、獲取偏差和付費用戶的計算限制” 4.6：迭代編碼任務的品質嚴重下降」#46099 來證明其結論。

然而，Margin Lab 的數據表明，Claude Opus 4.6 至少在 SWE-Bench-Pro 測試中保持了其分數。自2月以來進行的評估顯示出一些變化，但沒有實質變化。

Anthropic 沒有立即回應對克勞德品質問題發表評論的請求。 ®
