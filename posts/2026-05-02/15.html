15. 2026-05-01｜納德拉誓言「贏回粉絲」後，微軟發布了第一個重大更新

全文翻譯（繁體中文・香港）

Microsoft 透過 4 月 30 日的非安全性更新兌現了優先考慮 Windows 穩定性的承諾。

在補丁星期二之前，昨天的更新充滿了修復，其中包括一些針對 Windows 資源管理器的修復。

據微軟稱：“此更新提高了相關explorer.exe進程的可靠性，因此它們在關閉檔案總管視窗後會停止。”

然後還有“一般可靠性”修復。 「此更新帶來了基礎更改，以幫助提高 explorer.exe 的可靠性，包括登入時、與工作列功能表和任務視圖互動時、從檔案總管的快速存取中取消固定項目時等等。”

微軟還提高了裝置啟動後啟動應用程式的效能，並且在查看大容量檔案時的儲存效能變得更加快速。說到存儲，命令列中的最大 FAT32 格式大小限制已從 32GB 增加到 2TB：變化雖小，但生活品質確實得到了改善。

其他可靠性調整包括 Windows Hello、Microsoft Store 和工作列系統托盤的改進。

本週早些時候，微軟執行長薩蒂亞納德拉表示，該公司正在努力「贏回」粉絲。今年三月，Windows 老闆 Pavan Davuluri 承認 Windows 需要更快、更可靠，並包含更有針對性的人工智慧部署。他當時表示：“我們正在減少不必要的 Copilot 入口點，從截圖工具、照片、小部件和記事本等應用程式開始。”

此次更新兌現了 Davuluri 承諾的速度和可靠性，但人工智慧仍然是不可避免的。它至少應該比記事本（後來稱為“書寫工具”）的侵入性小。工作列能夠顯示 AI 代理的進度 - Microsoft 365 Copilot 應用程式中的研究人員是第一批和第三方應用程式的第一個採用者。

更新中還包括企業狀態漫遊 (ESR)，它允許用戶在裝置之間漫遊，並且現在可以透過 Windows Backup for Organizations 進行管理。還可以基於策略刪除預先安裝的 Microsoft 應用程序，並提高批次檔的安全性和效能。在後一種情況下，管理員可以為批次檔啟用更安全的處理模式，防止它們在執行過程中發生變更。

總而言之，這是一個有意義的更新，也許是 Windows 超級油輪正在慢慢改變航向的一個跡象。

但如果緊隨其後的是另一個匆忙的帶外補丁來修復修復程序，它所產生的善意將很快消失。 ®
