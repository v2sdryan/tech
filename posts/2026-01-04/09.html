9. 🔐 2026-01-04 資安討論：MCP 伺服器安全隱患 開發者須知的五大風險

原文連結：https://lihkg.com/thread/3894005/20260104

全文翻譯（繁中）

隨著 Model Context Protocol（MCP）在 AI 開發者社群中迅速普及，2026-01-04 連登資安版出現了一篇詳盡的 MCP 安全風險分析帖文。作者為資深滲透測試工程師，整理了五大常見安全隱患及其應對方案。

風險一：未加密傳輸。許多開發者在本地開發時使用 HTTP 而非 HTTPS 運行 MCP 伺服器，導致 AI 代理與工具之間的通訊可被中間人攔截。若 MCP 伺服器暴露在公網上，攻擊者可竊取敏感數據或注入惡意指令。建議：所有 MCP 通訊必須使用 TLS 加密。

風險二：缺少身份驗證。MCP 協議本身不強制要求身份驗證，許多第三方 MCP 伺服器預設開放訪問。攻擊者可直接連接未設防的 MCP 伺服器，存取其背後的資料庫或 API。建議：啟用 API Key 或 OAuth 2.0 認證，並限制允許的客戶端 IP 範圍。

風險三：過度授權。部分 MCP 伺服器授予 AI 代理過高的權限——例如資料庫的 DROP TABLE 權限或檔案系統的 root 存取權。一旦 AI 代理被提示注入（Prompt Injection）攻擊劫持，可能執行破壞性操作。建議：遵循最小權限原則，為 AI 代理建立專用的受限帳戶。

風險四：日誌洩漏。MCP 伺服器的日誌中可能包含敏感資訊——包括 API 密鑰、資料庫連接字串與用戶個人資料。若日誌未加密存放或被上傳至外部監控服務，可能造成數據洩漏。建議：實施日誌脫敏處理，敏感欄位自動替換為遮罩值。

風險五：供應鏈攻擊。社群貢獻的第三方 MCP 伺服器可能包含惡意程式碼——類似於 npm 套件的供應鏈攻擊。攻擊者可在看似正常的 MCP 伺服器中埋入後門，竊取經過的所有數據。建議：只使用經過審計的 MCP 伺服器，或自行構建並維護所需的 MCP 連接器。
