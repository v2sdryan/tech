01. 2026-03-18｜Linux 基金會開始努力保護 FOSS 維護者免受 AI slop bug 報告的影響

全文翻譯（繁體中文・香港）

六家大型科技公司共同向一個專案提供了 1,250 萬美元的資助，該專案旨在幫助開源專案的維護者應對 AI 錯誤報告。

「隨著安全情勢變得越來越複雜，人工智慧的進步正在極大地提高開源軟體中漏洞發現的速度和規模，」Linux 基金會關於該計畫的公告解釋道。 “維護人員現在面臨前所未有的大量安全問題，其中許多是由自動化系統生成的，而沒有有效分類和修復所需的資源或工具。”

Anthropic、AWS、GitHub、Google、微軟和 OpenAI 決定共同為該專案提供 1,250 萬美元的資助。

Alpha-Omega 是 Linux 基金會項目，致力於提高開源供應鏈的安全性，它將與開源安全基金會 (OpenSSF) 一起開展新的工作。

我們被告知這兩個組織“直接與維護人員及其社區合作，使新興的安全功能易於使用、實用，並與現有項目工作流程保持一致。”此外：“這項工作將支持可持續戰略，幫助維護人員管理不斷增長的安全需求，同時提高開源生態系統的整體彈性。”

Linux 基金會的聲明中包含了 Greg Kroah-Hartman 對 Linux 核心專案的一段原話，開頭是“僅靠撥款並不能幫助解決人工智慧工具如今給開源安全團隊帶來的問題。”

別擔心，溫柔的讀者，GKH 並沒有拋棄這個想法。引述繼續說道：“OpenSSF 擁有支持眾多項目所需的活躍資源，這些項目將幫助這些過度勞累的維護人員對他們當前收到的越來越多的人工智能生成的安全報告進行分類和處理。”

目前還沒有關於該項目具體將做什麼或何時發生的消息。

人工智慧產生的錯誤報告壓倒自由開源軟體維護者的問題並不新鮮。 Python 軟體基金會在 2024 年底對此提出了抱怨。最近，由於人工智慧產生的大量貢獻造成了困難，流行的開源資料傳輸工具 cURL 的維護者終止了該專案的錯誤賞金計劃。

就連微軟的 GitHub 也考慮對大量低品質、人工智慧產生的自由開源專案貢獻採取行動。 ®
