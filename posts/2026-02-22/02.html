02. 🧠 2026-02-22 AI 模型：Claude Opus 4.6 Agent Teams 功能革新多代理協作

全文翻譯（繁中）

Anthropic 於 2026 年 2 月 22 日正式推出 Claude Opus 4.6，這是該公司旗艦模型系列的最新版本，也是首個原生支援「Agent Teams」多代理協作功能的大型語言模型。根據 Anthropic 和 TechCrunch 的報導，Opus 4.6 支援高達一百萬 token 的上下文窗口，並在 Terminal-Bench 2.0 和 Humanity's Last Exam 兩大權威基準測試中同時奪得榜首，展現了其在代理式任務執行和深度推理方面的卓越能力。

Agent Teams 功能是 Claude Opus 4.6 的核心創新。在 Claude Code 環境中，開發者現在可以同時啟動多個 Claude 代理，每個代理擔任不同角色——例如架構師、代碼審查員、測試工程師和安全分析師——它們能夠並行工作、互相交流並協調各自的輸出。這種多代理協作模式讓複雜的軟件開發任務可以被分解為多個獨立子任務，由專門的代理分別處理後再整合結果。在實際測試中，Agent Teams 在處理大型重構任務時的效率較單一代理模式提升了三至四倍。

在基準測試方面，Claude Opus 4.6 的表現令人矚目。在 Terminal-Bench 2.0 測試中，該模型以 82.7% 的成績大幅領先第二名的 GPT-5.2（74.1%），展現了其在真實終端環境中執行複雜操作的卓越能力。在被稱為「人類最後考試」的 Humanity's Last Exam 中，Opus 4.6 取得了 41.3% 的正確率，首次突破 40% 大關，成為第一個在這項極限挑戰中展現出人類專家級推理能力的 AI 模型。百萬 token 的上下文窗口讓模型能夠在單次對話中處理約 75 萬個中文字符，相當於一整套技術文檔或一個中大型代碼庫。

Anthropic 同時宣佈了 Claude Code 的重大升級。除 Agent Teams 外，新版 Claude Code 引入了 Hooks 系統的強化版本，支援更精細的 PreToolUse 和 PostToolUse 控制，讓開發者能夠為每個代理操作設定自動化的品質檢查流程。新增的 Worktree 功能讓代理可以在隔離的工作環境中進行實驗性修改，不影響主分支代碼。此外，Claude Code 的 MCP（Model Context Protocol）整合也獲得大幅優化，支援更多第三方工具和數據源的無縫接入。

市場分析師認為，Claude Opus 4.6 的發佈標誌著 AI 行業正式進入「多代理時代」。Agent Teams 概念的引入不僅改變了開發者與 AI 互動的方式，更重新定義了軟件開發的工作流程。在定價方面，Anthropic 維持了具競爭力的策略，Opus 4.6 的 API 定價與前代持平，Agent Teams 功能對 Claude Code 的付費用戶免費開放。業界預計，隨著多代理協作模式的成熟，AI 輔助開發將從「一對一問答」進化為「團隊協作」，開發者的角色也將從「代碼撰寫者」轉變為「AI 團隊指揮官」。
