3. 🔧 2026-02-08 開發工具：Cursor 更新 Tab 自動補全與多檔案編輯能力

原文連結：https://www.cursor.com/changelog/2026-02-08

全文翻譯（繁中）

AI 程式碼編輯器 Cursor 於 2026-02-08 發佈重大更新，大幅提升 Tab 自動補全的準確度與多檔案編輯能力。新版本引入了「上下文感知補全」技術，AI 不僅理解當前檔案的程式碼結構，更能追蹤整個專案的型別定義、函數呼叫關係與設計模式。

Tab 補全的改進尤為顯著。在內部測試中，新版補全的首選接受率從 32% 提升至 48%，前三選接受率達到 71%。系統會自動分析最近的 Git 提交歷史，學習開發者的編碼風格與偏好命名慣例。對於 TypeScript 和 Python 專案，型別推斷的準確度提升了 40%。

多檔案編輯功能是本次更新的核心亮點。開發者現在可以在對話中描述一個橫跨多個檔案的修改需求，Cursor 會自動識別所有需要修改的檔案，生成差異預覽並一次性應用。系統會自動處理跨檔案的引用更新——例如修改一個函數的簽名後，所有呼叫處都會同步更新。

新增的 Agent Mode 讓 Cursor 可以自主執行多步驟開發任務。開發者可以發出「新增用戶登入功能」這樣的高階指令，Agent 會自動規劃實作步驟、建立路由、撰寫表單元件、整合認證服務並生成對應的測試案例。整個過程中開發者可以隨時介入調整方向。

社群回饋數據顯示，使用新版 Cursor 的開發者生產力平均提升約 40%。最大的改善出現在重構任務中——過去需要手動修改數十個檔案的大規模重構，現在可以在幾分鐘內透過 AI 自動完成。一位使用者分享道：「我用 Cursor 在 20 分鐘內完成了從 REST 到 GraphQL 的 API 遷移，這在過去至少需要兩天。」

Cursor 同時推出了團隊版方案，支援共享的 AI 上下文與編碼規範。團隊成員的補全建議會自動遵循專案的程式碼風格指南，確保 AI 生成的程式碼與人工撰寫的部分風格一致。月費定價 $40 每位開發者，包含無限量的 AI 補全與每月 500 次 Agent 任務。
