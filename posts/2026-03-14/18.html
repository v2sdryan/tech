18. 2026-03-13｜經過多年的堅持，ARM64 Linux 用戶終於獲得了 Chrome 日期

全文翻譯（繁體中文・香港）

Chrome 終於登陸 ARM64 Linux 設備，幾年後它出現在 Arm 上的 macOS 和 Windows 上。

谷歌在 Chromium 部落格上確認了即將發布的版本（將於 2026 年第二季度發布），並表示：“推出適用於 ARM64 Linux 設備的 Chrome 可以讓更多用戶享受谷歌最有用的服務與他們的瀏覽器的無縫集成。”

Chromium 早已可用於 Linux（以及各種其他平台），但是，在 Chrome 方面，ARM64 作業系統已經落後於其他作業系統，Chrome 將 Google 的服務新增至 Chromium 引擎。

從技術層面來看，這項宣布「也正是時候」。缺乏正式的 Chrome 版本是令人費解的遺漏，也是基於 ARM 的 Chromebook 群組管理員的痛點。登入 Google 帳戶並讓服務、設定和書籤跨裝置追蹤使用者的能力無疑很有吸引力。

然而，Chrome 的專有性質對於 Linux 社群的一些成員來說並不那麼有吸引力，也不需要擁有 Google 帳戶才能存取這家搜尋巨頭的許​​多產品。也就是說，考慮到 Chrome 目前擁有 73.26% 的市場份額（根據 Statcounter 的數據），安裝瀏覽器很誘人，即使幕後有熟悉的 Chromium 引擎。

谷歌寫道：「這項舉措滿足了人們對瀏覽體驗日益增長的需求，將開源 Chromium 計畫的優勢與Google的應用程式和功能生態系統結合。

“這個版本是一項重大任務，旨在確保 ARM64 Linux 用戶獲得與其他平台相同的安全、穩定和豐富的 Chrome 體驗。”

鑑於 x64 版本早已推出，並且將 Chrome 移植到其他基於 ARM64 的作業系統幾乎沒有出現任何技術問題，因此漫長的等待是值得注意的。正如 Vulture Central 的一位調侃者所指出的那樣，“有人在某個地方勾選了建造農場的方框…”

因此，問題是，谷歌為什麼現在要這樣做？ Chromebook 數量激增？或者，該公司可能正在尋求利用 Windows 難民，以逃避微軟及其硬體要求和遙測收集。在不同的作業系統中，還有什麼比 Chrome 更熟悉的呢？
