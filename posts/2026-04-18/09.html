09. Fri, 17 Ap｜駭客正在濫用未修補的 Windows 安全漏洞來侵入組織

全文翻譯（繁體中文・香港）

據一家網路安全公司稱，過去兩週，駭客利用一名心懷不滿的安全研究人員在網路上發布的 Windows 漏洞侵入了至少一個組織。

週五，網路安全公司 Huntress 在 X 上的一系列貼文中表示，其研究人員發現駭客利用了三個 Windows 安全漏洞，即 BlueHammer、UnDefend 和 RedSun。

目前尚不清楚這次攻擊的目標是誰，以及駭客是誰。

BlueHammer 是微軟迄今已修復的三個被利用漏洞中唯一的漏洞。 BlueHammer 的修復程序於本週稍早推出。

駭客似乎正在利用安全研究人員在線上發布的漏洞程式碼來利用這些漏洞。

本月早些時候，一位名為 Chaotic Eclipse 的研究人員在他們的部落格上發布了他們所說的利用 Windows 中未修補漏洞的程式碼。研究人員提到與微軟的一些衝突是發布程式碼的動機。

「我沒有嚇唬微軟，我又這麼做了，」他們寫道。他們補充說：「非常感謝 MSRC 的領導使這一切成為可能。」他們指的是微軟的安全響應中心，該中心是該公司負責調查網路攻擊並處理漏洞報告的團隊。

幾天后，Chaotic Eclipse 發布了 UnDefend，然後在本週早些時候發布了 RedSun。研究人員在其 GitHub 頁面上發布了利用所有三個漏洞的程式碼。

所有三個漏洞都會影響 Microsoft 製造的防毒軟體 Windows Defender，使駭客能夠獲得對受影響的 Windows 電腦的高級或管理員存取權限。

TechCunch 未能聯繫到 Chaotic Eclipse 置評。

在回答一系列具體問題時，微軟公關總監本·霍普(Ben Hope)在一份聲明中表示，該公司支持“協調漏洞披露，這是一種廣泛採用的行業實踐，有助於確保問題在公開披露之前得到仔細調查和解決，支持客戶保護和安全研究社區。”

這是網路安全產業所謂的「完全揭露」的案例。當研究人員發現缺陷時，他們可以將其報告給受影響的軟體製造商以幫助他們修復。那時，公司通常會確認收到，如果漏洞是合法的，公司會努力修復它。通常，公司和研究人員會商定一個時間表，確定研究人員何時可以公開解釋他們的發現。

有時，由於各種原因，溝通會中斷，研究人員會公開揭露該錯誤的詳細資訊。在某些情況下，部分為了證明缺陷的存在或嚴重性，研究人員會更進一步，發布能夠濫用該缺陷的「概念驗證」程式碼。

當這種情況發生時，網路犯罪分子、政府駭客和其他人就可以獲得程式碼並使用它進行攻擊，這促使網路安全防御者急於應對後果。
