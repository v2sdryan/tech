14. 2026-03-13｜憑證竊取人員欺騙來自 Cisco、Fortinet 和其他公司的 VPN 用戶端

全文翻譯（繁體中文・香港）

據微軟稱，一群被追蹤為 Storm-2561 的網路犯罪分子正在使用來自 CheckPoint、Cisco、Fortinet、Ivanti 和其他供應商的虛假企業 VPN 用戶端來竊取用戶的憑證。

Storm-2561 是一個新的犯罪團夥（「Storm」後跟數字是微軟追蹤仍在開發中的組織的方式），自 2025 年 5 月以來一直存在，通常使用 SEO 定位和冒充供應商來分發惡意軟體。這場從一月中旬開始的活動也不例外。

攻擊者透過操縱搜尋結果來獲得對受害者的初步訪問權限，並將偽裝成企業 VPN 更新的惡意網站推送到清單頂部。因此，當使用者搜尋「Pulse VPN 下載」或「Pulse Secure 用戶端」等 VPN 用戶端時，最上方的結果會指向模仿真實供應商頁面的欺騙網站。除了上面列出的 VPN 供應商之外，這些產品還包括 SonicWall、Sophos 和 WatchGuard 的產品。

點擊該連結會將使用者重新導向到惡意 GitHub 儲存庫，該儲存庫託管偽裝成 Microsoft Windows Installer (MSI) 檔案的虛假 VPN 用戶端。

雷德蒙德的威胁情报团队在周四的博客中表示：“微软已经观察到各种 VPN 软件品牌的欺骗行为，并观察到以下两个域的 GitHub 链接：vpn-fortinet[.]com 和 ivanti-vpn[.]org”。 GitHub 儲存庫現已刪除。 （請閱讀部落格的最後，了解一長串妥協指標。）

安裝程式在安裝過程中旁載入惡意動態連結程式庫 (DLL) 檔案、dwmapi.dll 和spector.dll，而虛假 VPN 軟體會提示使用者輸入其憑證。這會捕獲使用者名稱和密碼，然後將它們發送到攻擊者控制的命令和控制伺服器，同時看起來是合法的客戶端應用程式。

MSI 檔案和惡意 DLL 是使用太原利華近資訊科技有限公司提供的有效（現已撤銷）數位憑證進行簽署的。

接下來是最棘手的部分：使用者在虛假登入頁面輸入憑證後，應用程式立即顯示錯誤訊息，指出安裝失敗，然後指示受害者從供應商的官方網站下載合法的 VPN 用戶端。在某些情況下，該應用程式甚至會打開用戶的瀏覽器訪問合法網站。

該部落格寫道：“如果用戶隨後成功安裝並使用合法的 VPN 軟體，並且 VPN 連接按預期工作，則不會有任何跡象表明最終用戶受到損害。” “用戶可能會將初始安裝失敗歸因於技術問題，而不是惡意軟體。”

毫不奇怪，由於這是一份微軟威脅情報報告，這家軟體巨頭推薦其產品和服務來防止憑證盜竊。但我們要強調一些關鍵（且與供應商無關）的安全建議。

首先 - 我們必須強調這一點 - 對所有帳戶實施多重身份驗證 (MFA)。確保刪除被排除在 MFA 之外的用戶，並要求所有裝置隨時隨地進行 MFA。

第二：提醒員工不要將工作場所憑證儲存在使用個人憑證保護的瀏覽器或密碼庫中。 ®
