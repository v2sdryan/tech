1. 💻 2026-02-13 開發工具：Claude Code CLI 多代理協作模式實戰

原文連結：https://docs.anthropic.com/en/docs/claude-code/multi-agent-2026-02-13

全文翻譯（繁中）

Anthropic 於 2026-02-13 發佈 Claude Code CLI 的多代理協作模式更新，讓開發者可以在終端環境中同時協調多個 AI 代理執行不同任務。這一功能徹底改變了大型程式碼庫的管理方式。

多代理協作的核心概念是「分工」——主代理（Orchestrator）負責理解需求、制定計劃並協調工作；子代理（Workers）各自負責特定任務如程式碼審查、測試生成、文檔更新與安全掃描。所有代理共享專案上下文，但各自擁有專屬的工具集。

在實際測試中，多代理模式展現了驚人的效率。一位開發者分享了將 10 萬行 TypeScript 專案從 Express 遷移到 Hono 框架的案例：主代理規劃了遷移策略，一個子代理負責逐檔案改寫路由定義，另一個同步更新測試案例，第三個負責修正型別錯誤。整個遷移在 3 小時內完成，而人工估計需要 2-3 週。

Claude Code 的 Hooks 系統也獲得重要升級。開發者可以配置 PreToolUse 與 PostToolUse 鉤子，在每次工具呼叫前後自動執行自定義腳本——例如自動運行 Prettier 格式化、ESLint 檢查或 TypeScript 型別驗證。這確保了 AI 生成的程式碼始終符合團隊的編碼標準。

新增的「/plan」模式讓開發者在 AI 開始寫程式碼之前先審閱實作計劃。AI 會先分析需求、識別需要修改的檔案、列出潛在風險並提出實作方案。開發者可以調整計劃後再授權執行，大幅減少了返工的可能性。

Anthropic 同時公佈了 Claude Code 的使用統計：月活開發者已突破 100 萬，日均處理的程式碼修改超過 5,000 萬行。最受歡迎的使用場景依次為：重構現有程式碼（28%）、新增功能（24%）、除錯（19%）、撰寫測試（16%）與文檔生成（13%）。
